[HIGH-01] Python 3.14/3.15 Migration
  Status:  Not started — planned for future. Now on PyTorch (v5) — TF dependency
           removed from training pipeline. segment.py still uses PyTorch for
           inference. batch_axon and morphometrics have no ML dependencies.
  Action:  Full dependency audit required when migration begins. All unpinned
           packages need re-testing against new environment.

[HIGH-03] Dataset Split — Phenotype Balance
  File:    train/dataset/data_loader.py  →  load_all_patches()
  Issue:   Val/test split is currently random — no guarantee of 50% regen /
           50% control in the validation set. With a small dataset this can
           produce unbalanced validation metrics that don't reflect real
           model performance across phenotypes.
  Action:  Implement stratified split by image filename prefix (e.g. 'P' for
           pig/regen vs no prefix for control). Ensure val set always contains
           equal representation of both phenotypes.

[HIGH-04] Mask Annotation Protocol — Document
  Issue:   No formal guidance on how many masks to annotate for first training
           runs. Directly tied to batch size math and phenotype balance.
  Recommendations (0.2 val split, 63 patches/image, target bs=128):
    Minimum viable : 10 regen + 10 control = 20 total
                     → 16 train images → 1008 patches → bs=128 at 88% ✅
    Good           : 12 regen + 12 control = 24 total
                     → 20 train images → 1260 patches → bs=128 at 97% ✅
    Paper quality  : 18 regen + 18 control = 36 total
                     → 29 train images → 1827 patches → bs=128 at 27% ❌
                     → 32 train images → 2016 patches → bs=128 perfect ✅
  Priority annotation order:
    1. Regen images first (more morphological diversity)
    2. Different nerve types within each phenotype
    3. Different animals within each phenotype
  Action:  Add annotation guidance to README.md under a new Training section.

SECTION 3 — RESOLVED THIS SESSION (Session 3)
================================================================================

✅ PyTorch migration — full TF → PyTorch rewrite of training pipeline.
   smp.UnetPlusPlus, smp.losses, smp.metrics, manual training loop.
   Training confirmed running on A100.

✅ Logger overhaul — renamed console.py → logger.py. Per-line timestamps,
   _ts(), _append(), _append_raw(). write_section() with grep-able markers.
   finalize() computes elapsed time. context dict in header.

✅ Version system — utils/version.py created. __version__ = "5.0.0",
   __codename__ = "v5_pytorch". Imported in logger.py, shown in every log header.

✅ Log header / training setup split — header (what user configured) vs
   training setup (what model actually trained on). No duplication.

✅ Augmentation refactor — geometric_prob / photometric_prob split.
   augment_pair() returns applied list. augment_dataset_np() returns
   (images, masks, n_augmented, aug_counts). Config nested into
   augmentation.probabilities and augmentation.parameters.

✅ Augmentation log — shows n_augmented/total, geo_prob, photo_prob,
   per-transform counts split into Geometric and Photometric lines.

✅ Early stopping — changed from val_loss to combined axon+myelin dice.
   Primary trigger: combined > best_combined + EARLY_STOP_MIN_DELTA.
   Tiebreaker: same combined but val_loss improves — updates checkpoint,
   does NOT reset patience counter.
   checkpoint_flag: '' | ' ← CHECKPOINT' | ' ← CHECKPOINT (loss tiebreak)'

✅ Checkpoint summary — on_train_end() writes CHECKPOINT SUMMARY section
   to log: best epoch, axon dice, myelin dice, val loss, filename.
   Replaces redundant epoch metrics table.

✅ TrainingLogger — renamed on_epoch_end() → log_epoch(checkpoint_flag).
   Epoch line includes checkpoint flag inline. Rich table removed.

✅ Loss constants — DICE_WEIGHT, CE_WEIGHT, CE_SMOOTH from config.
   Loss function log line derived from constants, not hardcoded.
   Config: training.dice_weight, training.ce_weight, training.ce_smooth.

✅ Single global config load — _config = load_config() at module level
   in train.py. GEO_PROB, PHOTO_PROB, all training constants derived once.

✅ aug_prob fully removed — probabilities now in config. Removed from
   train_model() signature, TrainingLogger, __main__.py, helpers.py.

✅ Batch size calculator — full redesign.
   VRAM-based GPU detection (_get_gpu_batch_candidates()) — works for any
   GPU without name matching. Three-zone logic:
     ≥ 75% full  → acceptable
     25–75% full → excluded
     < 25% full  → trim (drop remainder, present as perfect fit option)
   Interactive menu with device info, ideal batch sizes, all options shown.
   Custom entry with recalculation and zone classification.
   val_fraction prompted before batch size — n_train_patches estimated
   correctly using val split. compute_batch_options() returns structured dict.

✅ Config additions — gpu_batch_candidates, cpu_batch_candidates,
   min_last_batch_fullness (0.75), danger_last_batch_fullness (0.25)
   all inside training block.

✅ Overlap handling — get_hann_compatible_step(patch_size) added to
   utils/helpers.py. Used in preprocess.py and segment.py.
   Overlap derived from step in logs — not hardcoded percentage.
   IMAGE PREPROCESSING and MASK PREPROCESSING lines show overlap and step.
   PREPROCESSING SUMMARY shows patch_size, overlap_pct, step.
   segment_dir() logs overlap and step.

✅ Preprocessing summary — added after mask loop in batch_process().
   Shows total images, masks, patches, patches per image, patch size,
   overlap, step, output directories.

✅ prepare_dataset() — .png added to mask extensions (fixes warning for
   PNG masks not matching TIF images).

✅ OpenCV warning suppression — cv2.setLogLevel(3) added in preprocess.py
   and segment.py. Suppresses Olympus proprietary TIFF tag warnings.

✅ morphometrics_bins floor — changed from 0 to 0.5 in config.json.

✅ EarlyStopping log description — "— triggered on axon+myelin dice"
   added to training setup dict.

✅ model_path — renamed from {model_name}_best.pt to {model_name}.pt.

✅ Prompt order — val_fraction now prompted before batch size in __main__.py.
   Order: GPU → images folder → magnification → model name → epochs →
   val fraction → augmentation → batch size menu → log setup → run.
