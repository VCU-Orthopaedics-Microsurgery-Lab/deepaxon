  Issue:   No formal guidance on how many masks to annotate for first training
           runs. Directly tied to batch size math and phenotype balance.
  Recommendations (0.2 val split, 63 patches/image, target bs=128):
    Minimum (40X)  : 5 regen + 5 control = 10 total
                     → 8 train images → 504 patches → bs=128 at 94% ✅
    Minimum (100X) : 10 regen + 10 control = 20 total
                     → 16 train images → 1008 patches → bs=128 at 88% ✅
    Good (100X)    : 12 regen + 12 control = 24 total
                     → 20 train images → 1260 patches → bs=128 at 97% ✅
    Paper quality  : 18 regen + 18 control = 36 total
                     → 32 train images → 2016 patches → bs=128 perfect ✅
  Priority annotation order:
    1. Regen images first (more morphological diversity)
    2. Different nerve types within each phenotype
    3. Different animals within each phenotype
  Action:  Add annotation guidance to README.md under a new Training section.

SECTION 9 — V2 PRODUCTION MODEL (future)
================================================================================

These are architectural and preprocessing improvements to consider for the
next major model version after the current 40X rabbit study is complete.
None of these should be implemented mid-study as they require full retraining.

[V2-01] Z-score normalization per patch
  Current: L2 normalization along axis=1 (per row)
  Issue:   Patches from different staining intensities remain incomparable
  Fix:     Replace with (patch - mean) / std per patch
  Impact:  Must update both data_loader.py and segment.py simultaneously
           All existing models become incompatible — full retrain required

[V2-02] Stain normalization preprocessing
  Current: No inter-image staining normalization
  Issue:   Different animals, sessions, and staining batches produce
           different absolute intensities that the model must generalize across
  Fix:     Macenko or Vahadane stain normalization in preprocess.py before
           patchification — normalize all images to a reference image
  Impact:  Preprocessing step only, no architectural changes required
           Reference image should be a high-quality control nerve image

[V2-03] Instance normalization in encoder
  Current: Batch normalization in resnet34 encoder
  Issue:   Batch norm statistics are computed across the batch — sensitive
           to inter-image intensity variation
  Fix:     Replace batch norm with instance norm in encoder layers
           Makes model inherently invariant to global intensity shifts
  Impact:  Architectural change — full retrain required

[V2-04] Channel attention (CBAM or SE block)
  Current: No attention mechanism in UNet++ decoder
  Issue:   Model weights all feature channels equally regardless of
           local image context (staining, contrast regime)
  Fix:     Add Squeeze-and-Excitation or CBAM attention blocks to encoder
           Model learns which channels are informative per image context
  Impact:  Architectural change — full retrain required

[V2-05] Per-image statistics as conditioning input
  Current: Model has no explicit awareness of global image statistics
  Issue:   Patches from different images treated identically regardless
           of source image staining characteristics
  Fix:     Compute per-image mean, std, contrast, skewness — append as
           additional input channels or conditioning vectors
  Impact:  Architectural change — full retrain required

[V2-06] Contrastive pretraining of encoder
  Current: ImageNet pretrained resnet34 encoder
  Issue:   ImageNet features are not optimized for peripheral nerve histology
  Fix:     Contrastive pretraining on nerve patches — same image = positive
           pairs, different images = negative pairs — before segmentation
           fine-tuning. Teaches encoder global image identity awareness.
  Impact:  Significant training pipeline addition — pretrain then fine-tune

[V2-07] Two-stage cascade architecture
  Current: Single UNet++ model
  Issue:   Systematic errors (vessel wall oversegmentation) are hard to
           eliminate with annotation alone at small dataset sizes
  Fix:     Stage 1 — current segmentation model
           Stage 2 — correction model trained on (Stage 1 prediction,
           ground truth) pairs. Learns to fix systematic Stage 1 errors.
  Note:    Only pursue after annotation route is exhausted (50+ images,
           val_ax consistently >0.93, val_my >0.88)

[V2-08] Augmented rotation range
  Current: ±15° rotation
  Proposed: ±45° or full ±180°
  Rationale: Nerve cross-sections are rotationally symmetric — no preferred
             orientation exists. Larger rotation increases effective dataset
             diversity significantly.

[V2-09] Increased noise augmentation
  Current: noise_sigma=0.02 (nearly imperceptible)
  Proposed: sigma=0.05 to 0.08
  Rationale: Forces model to rely on structural features rather than
             absolute pixel intensities. Improves generalization across
             imaging sessions with different camera noise profiles.

[V2-10] Boundary loss component
  Current: Weighted Dice + CrossEntropy
  Proposed: Add boundary loss term for late training refinement
  Rationale: Vessel wall oversegmentation is a boundary precision problem.
             Boundary loss penalizes errors at class boundaries specifically.
  Note:    Switch from Dice+CE to Dice+CE+Boundary after warmup epochs only.
           Only pursue after data annotation route is exhausted.
