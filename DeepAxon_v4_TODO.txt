[CRIT-01] Train/Inference Normalization Mismatch
  File:    segment/segment.py  →  segment_image()
  File:    train/data/data_loader.py  →  load_patches()
  Issue:   keras.utils.normalize performs L2 normalization (divides by vector
           norm). load_patches() normalizes by /= 255.0 (min-max to [0,1]).
           These are different operations — train/inference mismatch means
           model receives differently scaled inputs at inference than it saw
           during training.
  Action:  Verify which normalization was used during original training.
           If min-max (likely): replace normalize(patch) in segment_image()
           with patch.astype(np.float32) / 255.0
           If L2: update load_patches() to use keras.utils.normalize instead
           of /= 255.0. Document the decision in both files with a comment.

[CRIT-02] BGW Color Mapping Mismatch
  File:    morphometrics/morphometrics.py  →  get_morphometrics()
  Issue:   axon_mask = cv2.inRange(img, 200, 255) captures white pixels.
           But segment/segment.py recolor() maps: grey(128)=axon,
           white(255)=myelin. So axon_mask is actually capturing myelin,
           not axons. The comment "white = axon" contradicts segment.py.
  Action:  Verify against actual segmented images. Expected correct masks:
             axon_mask  = cv2.inRange(img, 100, 180)  # grey = axon
             fibre_mask = cv2.inRange(img, 1, 255)    # all non-black
           Add cross-reference comment in both recolor() and get_morphometrics()
           stating the BGW contract explicitly.

[CRIT-03] NameError in prepare_dataset() — Will crash before training starts
  File:    train/train.py  →  prepare_dataset()
  Issue:   Return dict references cropped_img and cropped_mask but these
           variables are never defined in the function scope.
  Action:  Add after patches_img/patches_mask definitions:
             cropped_img  = images_dir / "cropped"
             cropped_mask = masks_dir  / "cropped"

[CRIT-04] Wrong Magnification Passed to batch_process()
  File:    train/train.py  →  train_model()
  Issue:   batch_process() called with mag=model_type, passing 'unet++' as
           the magnification string. patch_size lookup fails silently and
           falls back to default 256 — accidentally correct but semantically
           wrong and fragile.
  Action:  Add a mag parameter to train_model() signature. Prompt for
           magnification in train/__main__.py using a selection menu
           ('40X' / '100X') and pass through correctly to batch_process().

[CRIT-05] Duplicate Axon Assignment Not Implemented
  File:    morphometrics/morphometrics.py  →  get_morphometrics()
  Issue:   Module docstring states duplicate assignments are removed but the
           matching loop has no mechanism to track assigned axon labels.
           Multiple fibres can silently match the same axon, inflating counts
           and corrupting g-ratio measurements.
  Action:  Add before the fibre loop:
             assigned_axon_labels = set()
           After get_axon_row() returns a match, check:
             axon_label = int(axon_row['label'].iloc[0])
             if axon_label in assigned_axon_labels:
                 continue
             assigned_axon_labels.add(axon_label)

[CRIT-06] Mask Resize Interpolation Bug
  File:    utils/resize.py  →  resize_img()
  Issue:   is_mask parameter is accepted but ignored. cv2.resize() has no
           interpolation argument so both images and masks use OpenCV default
           INTER_LINEAR. For binary masks, INTER_LINEAR interpolates between
           0 and label values, corrupting mask labels.
           This cascades into train/data/preprocess.py where is_mask=True
           is passed but has no effect.
  Action:  Change cv2.resize call to:
             interpolation = cv2.INTER_NEAREST if is_mask else cv2.INTER_AREA
             img = cv2.resize(img, TARGET_SIZE, interpolation=interpolation)
           Also fix docstring which claims INTER_AREA but no flag was set.
           preprocess.py fix is automatic once resize.py is corrected.

[CRIT-07] Mask Values Not Converted to Class Indices Before Training
  File:    train/data/data_loader.py  →  load_patches()
  File:    train/train.py  →  train_model()
  Issue:   Masks loaded as raw float32 with values 0.0, 128.0, 255.0.
           These are passed directly to model.fit() with softmax output and
           combined_loss. Loss function expects integer class indices (0,1,2)
           or one-hot encoding — raw 0/128/255 values will produce incorrect
           loss calculations.
  Action:  Verify combined_loss handles raw values. If not, add conversion
           after loading: Y = np.round(Y / 128).astype(np.float32)
           which maps 0→0, 128→1, 255→2 (rounded). Document the decision.

[CRIT-08] ROI Coordinate Space — Potential Mismatch
  File:    batch_axon/overlay/export_roi.ijm
  File:    batch_axon/overlay/process_overlay.py  →  get_overlay_area()
  Issue:   The macro crops the original resolution image (e.g. 2880×2048)
           before exporting ROI coordinates. process_overlay.py then
           rasterizes those coordinates onto an image opened via PIL at its
           stored resolution. If the CSA .tif is stored at original resolution
           but segmented outputs are resized to 1440×1024, coordinate spaces
           may not align.
  Action:  Confirm what resolution the CSA overlay .tif files are saved at
           by Fiji. Ensure the image opened in get_overlay_area() is the same
           resolution as the coordinate space the macro exports. Add a comment
           documenting the confirmed resolution assumption.
